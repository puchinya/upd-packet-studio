# UDP Packet Studio (v0.1.0)

UDP Packet Studio は、Rust と `egui` / `eframe` を使用して構築された、高性能でモダンなデザインのデスクトップ向け UDP パケット送受信・解析スタジオ（ツール）です。 ECHONET Lite プロトコルの解析補助やマルチキャスト送受信、詳細な Wireshark 風のヘキサデシマル・インスペクター、非同期のログ自動保存機能などを備えています。

---

## 🚀 主な機能

### 1. 📁 パケットコレクション管理 (Collections)
- よく使用するパケット定義（送信先IP/ポート、データ形式、ペイロードなど）をコレクション（フォルダ）単位で管理・整理・保存できます。
- ワンクリックでコンポーザー（送信機）に反映させたり、ダイレクトに送信することができます。

### 2. 🚀 パケットコンポーザー (Composer / Sender)
- 宛先ターゲットアドレス（IP:Port）の指定、ペイロードの指定（Text 形式 / Hex 形式）をしてパケットを送信できます。
- **ECHONET Lite 送信支援ヘルパー**: ECHONET Lite プロトコルのヘッダー構造（TID, SEOJ, DEOJ, ESV, EPC, EDT）をUIから直感的に構築し、ワンクリックで送信パケットを生成する機能が統合されています。

### 3. 📊 リアルタイム・ログモニター (Logs / Log Viewer)
- 送受信（SENT / RECV）したすべてのパケット情報、およびソケット結合（Bind）エラーなどのシステム情報をリアルタイムにリスト化します。
- IPフィルター、オートスクロール機能、クリア機能を搭載しています。
- 蓄積したログデータは、手動で **CSV**, **JSON**, **PCAP (Wireshark 互換)** 形式にエクスポートが可能です。

### 4. 🔍 パケットインスペクター (Inspector)
- ログからパケットを選択し、詳細な内部バイナリ構造を調査できます。
- **ヘキサダンプ (Hex Dump)**: Wireshark 風の行オフセット、16進表現、ASCII表示。
- **プレーンテキスト (ASCII)**: 文字列パケットのプレーン表示。
- **ECHONET Lite デコーダー**: ECHONET Lite 規格に基づき、トランザクションIDやサービスコード、プロパティ（EPC/EDT）などを詳細にパース・解説表示します。

### 5. 🌐 マルチキャスト送受信 (Multicast)
- 指定のネットワークインターフェースから、IPv4 および IPv6 のマルチキャストグループ（例: `224.0.23.0` 等）へ参加（Join）および離脱（Leave）し、マルチキャストパケットを受信できます。

### 6. ⚙️ ログ自動保存機能 (Auto-Save Logs)
- 設定ダイアログから、パケットログの非同期自動保存を設定できます。
- UIスレッドを一切ブロックせず、バックグラウンドのワーカースレッドが安全にファイルへ追記します。
- **自動保存フォーマット**: CSV、JSON (JSON Lines/NDJSON)、PCAP (パケットキャプチャ) から選択可能。
- **デフォルト保存先**: `~/UdpPacketStudio/Logs` (設定画面から任意のフォルダにブラウズ・変更可能)。

## 🎨 UI デザインシステム
- **モダン・ダークテーマ**: 長時間の作業でも目が疲れない、コントラストが考慮されたプレミアムダークカラースキーマ。
- **Mac風タイトルバー**: タイトルバー領域でのドラッグ移動、ダブルクリックによる最大化、3色丸ボタンによる閉じる/最小化/最大化ウィンドウ制御。
- **レスポンシブ・ドッキング**: `egui_dock` によるタブ分割レイアウトにより、Collections、Composer、Logs、Inspector、Multicast の各パネルを自由な配置で並べて同時監視できます。
- **ステータス表示**: ソケットのアクティブ状況（🟢 Active / 🔴 Offline）、バインド中のポート、ロギングされたパケット数、ログ自動保存の動作形式などを最下部に常時表示。

## 🛠 ビルドおよび動作方法

### 動作環境
- **Rust コンパイラ**: Rust 2024 エディション以降が推奨されます。

### ビルド & 実行手順
1. 本レポジトリをクローンします。
2. 以下のコマンドを実行してアプリケーションを起動します：
   ```bash
   cargo run --release
   ```
3. テストを実行するには、以下を使用します：
   ```bash
   cargo test
   ```

## 📂 ディレクトリ構成

- `src/main.rs`: アプリケーションのエントリーポイント、Mac風ウィンドウ外枠、UIイベントループ、および非同期ロガースレッド。
- `src/config.rs`: アプリ設定の永続化管理 (`updexp_config.json`)。
- `src/types.rs`: パケットログ、ペイロード種別、ログ形式、および PCAP 構造・ECHONET Lite 解析ヘルパーの共通型定義。
- `src/udp_worker.rs`: バックグラウンドスレッドでの UDP ソケットの Bind / Send / Recv と非同期メッセージング。
- `src/styling.rs`: ダークテーマ定義、角丸 (Corner Radius), 余白調整などの統一デザインシステム。
- `src/views/`: 各機能コンポーネントごとのUI描画ロジック。
  - `collections.rs`: コレクションツリーと定義の新規作成・編集UI。
  - `sender.rs`: 送信機および ECHONET Lite 送信アシスタントUI。
  - `log_viewer.rs`: 送受信ログの一覧テーブルと手動保存UI。
  - `inspector.rs`: 3通りのパケット詳細解析パネルUI。
  - `multicast.rs`: マルチキャストグループの参加管理UI。
